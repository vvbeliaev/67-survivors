# 67-survivors — Prototype Design (v0.2)

Дата: 2026-05-05. Статус: scope для рабочего прототипа.
Ссылка на лор/концепт: [CLAUDE.md](../../../CLAUDE.md).

## Цель прототипа

Играбельный online co-op (1–4 игрока), вся кор-логика, примитивная геометрия (без ассетов). Задача — валидировать петлю «4 класса × волны × апгрейды × финальный босс» и убедиться, что сетевая модель тянет.

## Стек и сетевая модель

- **Engine:** Godot 4.6, GDScript, GL Compatibility (уже инициализировано).
- **Net:** ENet, **host-authoritative**. Хост одновременно играет и держит мировое состояние. `MultiplayerSpawner` + `MultiplayerSynchronizer` для авто-репликации сущностей.
- **Authority:** ввод игроков — клиент-локальный (peer-authority на player-ноде); враги/волны/XP/боссы — host-authority.
- **Lobby:** простое меню Host/Join (адрес+порт+ник+класс), стартовая сцена `lobby.tscn`. Старт раунда — кнопкой хоста, после чего грузится `arena.tscn`.

## Сцены

```
scenes/
  lobby.tscn       # меню коннекта + выбор класса + список лобби
  arena.tscn       # игровой раунд (мир, спавнеры, HUD)
  player.tscn      # базовый игрок (CharacterBody2D)
  enemy.tscn       # базовый враг (CharacterBody2D)
  projectile.tscn  # снаряд (Area2D)
  hud.tscn         # CanvasLayer: HP/MP бары, таймер, лвл, выбор апгрейда
```

## Скриптовая структура

```
scripts/
  network.gd            # autoload: host/join/leave (есть)
  game_state.gd         # autoload: текущий ран — таймер, лвл пати, XP, фаза

  classes/

    mage.gd
    bard.gd
```

## Управление (как в концепте)

WASD движение. ЛКМ — основной скилл. ПКМ — второй скилл. Space — класс-утилка. Авто-атака идёт фоном.

## Камера

`centroid_camera.gd` (Camera2D, `top_level=true`, current на каждом клиенте):
- позиция = среднее координат живых игроков.
- зум = `clamp(base / spread, min_zoom, max_zoom)` где `spread` — макс. расстояние от центроида.
- если игрок отходит от центроида дальше `LEASH_RADIUS` (например 600px) — упругая стяжка (приклеивает velocity к центру). Расходиться нельзя — этот лиш = «экран сжимает».

Все 4 класса с базовой механикой. Архетипы и классовое дерево — Tier 2.

| Класс | HP | Speed | Авто-оружие | ЛКМ | ПКМ | Space | Ресурс |

| Маг | 90 | 240 | Магический снаряд (1.0с, 8 урона, лок на ближайшего) | Файрбол по курсору (8 урона AoE R=80, 30 маны, 0.8с КД) | Молния-цепь (3 цели, 18 урона, 50 маны, 4с КД) | Блинк по курсору (макс 220px, 5с КД, без маны) | mana 100, реген 8/с |

## Враги

Host-authoritative. Спавнятся `WaveDirector`, выбирают цель = ближайший живой игрок, простая steer + collision avoidance.

| Тип | HP | Speed | Урон | Атака | Поведение |
|---|---|---|---|---|---|
| Rusher | 25 | 180 | 8 | Контакт (cooldown 0.6с) | Прёт в ближайшего |
| Ranged | 18 | 110 | 6 | Снаряд (R=320, КД 1.5с, speed 240) | Держит дистанцию ~250px, стреляет |
| Tank | 120 | 90 | 18 | Контакт (КД 1.0с) | Медленно прёт, тяжёлый |
| Boss | 4000 | 100 | 30 | Контакт + телеграф-AoE (R=180, КД 6с, виндап 1.2с) | Медленный преследователь |

## Волны (`wave_director.gd`)

Таймер раунда: **10 минут (600с)** для прототипа. Шкала сложности по времени:

- 0:00–2:00 — только rusher, спавн каждые 2.5с по 2 шт. 
- 2:00–5:00 — +ranged, спавн каждые 2.0с по 3 шт. Каждые 60с волна-всплеск (10 шт сразу).
- 5:00–8:00 — +tank, спавн каждые 1.8с по 3–4. 
- 8:00–10:00 — все типы, спавн каждые 1.4с по 4–5.
- 10:00 — спавн стопается, спавнится **boss**. Win = boss мёртв.

Враги спавнятся в кольце радиусом 700–900px от центроида (за камерой).

## XP / Уровень пати / Апгрейды

- Каждый убитый враг даёт XP: rusher 1, ranged 2, tank 5, boss 50.
- XP — **пати-общий**. Порог уровня: `100 + 50 * level`.
- На лвл-ап: время **не** ставится на паузу (multiplayer), но каждому игроку показывается окно с 3 случайными апгрейдами (rpc на каждого пира). Игрок выбирает — апгрейд применяется к нему.
- Если игрок не успел/мёртв — апгрейд сгорает (просто не применяется).

**Пул апгрейдов (общий для всех классов, MVP):**

| ID | Эффект |
|---|---|
| max_hp | +20 max HP, heal 20 |
| move_speed | +10% движения |
| damage | +10% к исходящему урону |
| atk_speed | +10% к скорости авто-атаки |
| range | +10% к рейнджу/радиусам |
| cooldown | −8% к КД активных скиллов |
| regen | +1 HP/с |
| lifesteal | 5% от урона возвращается HP |
| mana_cap | +25 max mana (no-op для безманных) |
| mana_regen | +3 mana/с (no-op для безманных) |

3 случайных из пула, без повторов в одном выборе. Стакаются между уровнями.

## Смерть и респавн

- HP ≤ 0 → игрок **downed**: невидим для врагов, не двигается, не кастует. Виден остальным как «тело».
- Через **30с** автореспавн на позиции центроида с 50% HP.
- Все downed одновременно → **wipe** → сцена `lobby.tscn`.

L4D-стайл активного поднятия — Tier 2.

## HUD

CanvasLayer (на клиенте, не реплицируется):
- Левый верх: ник + class, HP/MP бары.
- Правый верх: таймер раунда, уровень пати, XP-бар.
- Низ: иконки 3 скиллов с КД-оверлеями (текстовые буквы Q/E/Space сойдут — это прототип).
- Центр-экран при лвл-апе: 3 кнопки апгрейдов.
- Экран wipe / win — текст по центру + кнопка «В лобби».

## Что НЕ делаем в прототипе

- Архетипы / класс-деревья → Tier 2.
- Кросс-классовые синергии → Tier 2.
- Ауры близости → Tier 2.
- Бард QTE → Tier 2 (заменено обычным кастом).
- Активное поднятие союзника → Tier 2 (только автореспавн).
- Несколько арен / переходы → Tier 2.
- Мета-прогресс / косметика → out of scope.
- Звук / ассеты → out of scope.
- Матчмейкинг / NAT punching → ручной ввод IP.
- Anti-cheat / валидация — host-trust модель, никакой защиты.

## Smoke-тест приёмки

1. `make import` без ошибок.
2. Запуск 2 экземпляров локально → host + join → оба видят друг друга в арене.
3. Каждый класс играбелен, авто-атака работает, ЛКМ/ПКМ/Space действуют.
4. Враги спавнятся, агрятся на ближайшего, наносят урон.
5. Лвл-ап → у обоих игроков всплывает 3-кнопочный выбор → пик применяется.
6. Игрок с HP≤0 → downed → респавн через 30с.
7. На 10:00 — спавн боссa, kill = win-экран.
8. Все мёртвы одновременно → wipe → лобби.

## Acceptance subagent

После имплементации запускается специализированный агент-приёмщик. Получит этот док и git diff, проверит:
- сцены/скрипты собираются (`make import`),
- структура соответствует спеке,
- основные RPC и authority корректны (нет двойной симуляции врагов),
- smoke-тест из секции выше выполняется (по коду + headless run).
