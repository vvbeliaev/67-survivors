# Mage Legendary — Echo Clone

Дата: 2026-05-08. Статус: дизайн (контент-спек, выезжает на инфраструктуре из rarity-spec).
Связанные доки: [rarity-upgrades-design.md](2026-05-08-rarity-upgrades-design.md).

## Требование (от пользователя)

> После блинка на месте из которого был блинк — создаётся неуязвимый недвижимый клон, который повторяет следующие 3 заклинания мага в ближайшие для копии цели.

## Расширение в механику

**Эффект легендарки.** Пассивно модифицирует Space-скилл мага (`mage_blink`):

1. При следующем (и каждом последующем) использовании блинка маг оставляет на исходной позиции **Echo Clone**.
2. Клон неуязвим (не получает урона, нельзя застанить, нельзя отбросить, не блокирует движение).
3. Клон неподвижен — стоит ровно на той точке, где маг был перед блинком.
4. Клон следит за каждым последующим **активным заклинанием** мага (ЛКМ — `mage_fireball`, ПКМ — `mage_chain`). Когда маг кастит такое заклинание, клон через 0.10s повторяет его, прицеливаясь по своей логике (см. ниже).
5. После того как клон повторил **3 заклинания**, он растворяется (визуальный fade ~0.4s).
6. Если маг блинкнул снова до того, как старый клон отыграл свои 3 повтора, старый клон растворяется немедленно, новый встаёт на новой исходной позиции с полным счётчиком 3. (Один клон в любой момент времени.)

**Что считается «заклинанием» для счётчика.**
- ✅ ЛКМ (fireball) — повторяется.
- ✅ ПКМ (chain lightning) — повторяется.
- ❌ Автоатака (mage_auto_bolt) — НЕ повторяется. Иначе клон превращается в DPS-турель, что обесценивает направленные заклинания. Авто — тонкий шум, не «заклинание».
- ❌ Space (blink) — NOT повторяется (клон всё равно неподвижен).

То есть клон ловит ровно 3 нажатия ЛКМ/ПКМ.

**Прицеливание клона.** Клон не следует за курсором мага — у клона своя логика «ближайшая цель»:

- Для fireball: нацелен в позицию ближайшего живого врага в радиусе досягаемости fireball'а от клона (если врагов в радиусе нет — кастит в направлении ближайшего за пределами радиуса; если врагов вообще нет — счётчик не тратится, клон ждёт).
- Для chain: первая цепочка-цель = ближайший враг от клона; дальше chain работает как обычно.
- Если в момент повтора подходящих целей вообще нет на арене — счётчик не тратится, клон ждёт следующего каста мага.

**Урон, манакост, кулдаун.** Повтор клона:
- Стоит **0 маны** (клон не имеет своей маны).
- Не идёт через CD скилла мага (клон — отдельный кастер).
- Использует текущие статы мага на момент **повтора**, не на момент исходного каста (`dmg_mult`, `range_mult`). Это даёт право легендарке скейлиться вместе с прокачкой.

## Тех-дизайн

### Класс `EchoClone` (новый узел)

Файл: `src/skills/concrete/mage_echo_clone.gd` (или `src/player/echo_clone.gd` — решит план реализации).

`extends Node2D`. Поля:

- `owner_player: Node` — маг.
- `repeats_left: int = 3`.
- `position: Vector2` — фиксирована на спавне.
- `lifetime_max: float = 25.0` — жёсткий хард-кэп: если за 25с не отыграл, растворяется. Анти-soft-lock.
- `pending_repeats: Array` — очередь повторов в формате `{kind: StringName, scheduled_at: float}` для отложенного 0.10s повтора.

Методы:

- `setup(player, pos)` — host-only.
- `on_player_cast(kind: StringName)` — добавляет в `pending_repeats` со временем `now + 0.10`. Только для kind ∈ {fireball, chain}.
- `_process(delta)` — на host'е тикает `pending_repeats`: когда `now >= scheduled_at` и есть подходящая цель — кастит и декремент `repeats_left`. Когда `repeats_left == 0` или `lifetime > lifetime_max` — `queue_free()` с пред-уведомлением для fade.

Репликация: клон — это spawn-сущность через `MultiplayerSpawner` арены (надо посмотреть, есть ли такой; если нет — свой). Клиенту нужны: `position` (spawn-once), `repeats_left` (для UI-индикатора, on_change), `state` (alive/fading, on_change).

Визуал: полупрозрачный синеватый силуэт мага, slight glow, цифра `repeats_left` над головой. Прорисовкой займётся отдельный View-узел по аналогии с `player_view.gd`.

### Хук в `mage_blink`

Файл: `src/skills/concrete/mage_blink.gd`.

В `on_pressed()` после успешного блинка:

```gdscript
var leg = owner_player.legendary_id
if leg == &"mage_echo_clone":
    _spawn_or_replace_clone(from_pos)
```

`_spawn_or_replace_clone(pos)` сносит старый клон если есть, спавнит новый, регистрирует его как «активный» у игрока (`owner_player.echo_clone = clone`).

### Хук в `mage_fireball` и `mage_chain`

В `on_pressed()` после реального каста (то есть после `consume_cost()` и `start_cooldown()`) — если у игрока активен клон, дёрнуть `clone.on_player_cast(kind)`.

Это один-два лишних `if` в каждом скилле. Альтернатива — глобальная подписка через EventBus (`spell_cast.emit(player, kind)`) — но это шире и трогает больше кода. Я выбрал точечный путь.

### Identification клона как «своего»

Клон-fireball/clone-chain должны:
- Не давать xp/lifesteal владельцу (или давать? решено: **не давать**, иначе соло-кастинг превращается в lifesteal-петлю).
- Урон записывать как `team = "player"` (как обычно), но без owner_player на проджектайле, чтобы lifesteal не сработал.

Реализация: переиспользуем `_spawn_projectile()` из `Skill`, но **не** передаём `owner_player` в проджектайл (или передаём `null` в специальном поле). Проверить нужно как именно lifesteal сейчас читается — если из owner'а проджектайла, то достаточно null. Если глобально на скилл-аутпуте — добавить флаг.

### Поле `Player.legendary_id`

В `Player` добавляем:

```gdscript
@export var legendary_id: StringName = &""
```

Реплицируется как spawn-only (легендарка выбирается один раз и не меняется).

В `apply_upgrade_def()`: если `def.rarity == LEGENDARY`, выставить `legendary_id = def.id`. Никакой особой логики, просто запоминаем что выбрана.

Скиллы потом сами читают `owner_player.legendary_id` и решают, активировать ли свою легендарную ветку.

### `UpgradeDef` для Echo Clone

Файл: `resources/upgrades/legendary_mage_echo_clone.tres`.

```
id = &"mage_echo_clone"
display_name = "Эхо"
description = "После блинка остаётся неуязвимый клон.\nКлон повторяет 3 следующих заклинания\n(fireball, chain) в ближайших к нему врагов."
flavor = "Тень мага задерживается там, где он был."
rarity = 3   # LEGENDARY
class_filter = Array[StringName]([&"mage"])
category = &"utility"

# stat / mode / amount — пустые, т.к. чисто механика, без числового стата
```

Иконка — выбрать с game-icons.net в духе «эхо/двойник/призрак» (например, `lorc/ghost-ally` или `lorc/echo-ripples`). На план реализации.

## Что НЕ входит

- Балансировка (3 повтора vs 5; задержка 0.10s vs 0.0s) — итерируем после первого тестового билда.
- Поведение при смерти мага во время активного клона — клон тоже растворяется (тривиально через `_process` → проверка `owner_player.alive`).
- Поведение при смене класса (его нет в забеге) — irrelevant.
- Звук/анимация прорастания клона — последующая полировка.

## Acceptance criteria

- На 10м уровне маг видит легендарку «Эхо» среди 3 опций.
- После пика: при следующем блинке создаётся клон.
- Клон неуязвим: получает 0 урона при попадании.
- Клон неподвижен: даже при толчках/ауре врагов не сдвигается.
- При следующем нажатии ЛКМ маг кастит fireball, через ~0.1s клон кастит fireball в ближайшего к себе врага.
- Аналогично для ПКМ (chain).
- Через 3 повтора клон растворяется.
- Если маг блинкнул второй раз до отыгрыша 3 повторов — старый клон исчезает, новый ставится со счётчиком 3.
- Автоатаки мага НЕ повторяются клоном.
- Клон не даёт мага лайфстил/ману за свои хиты.
